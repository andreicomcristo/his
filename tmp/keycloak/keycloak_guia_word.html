Guia de Configuracao Keycloak no HIS

1. Como subir o Keycloak com Docker

Para ambiente local, a forma mais simples e padrao e usar Docker.

Comando:

docker run --name keycloak-his ^
  -p 8081:8080 ^
  -e KC_BOOTSTRAP_ADMIN_USERNAME=admin ^
  -e KC_BOOTSTRAP_ADMIN_PASSWORD=admin ^
  quay.io/keycloak/keycloak:26.1.0 start-dev

No macOS/Linux, o mesmo comando fica:

docker run --name keycloak-his \
  -p 8081:8080 \
  -e KC_BOOTSTRAP_ADMIN_USERNAME=admin \
  -e KC_BOOTSTRAP_ADMIN_PASSWORD=admin \
  quay.io/keycloak/keycloak:26.1.0 start-dev

Depois disso, acessar:

http://localhost:8081

Login inicial:

usuario: admin
senha: admin

2. O que precisa configurar no Keycloak

O foco principal nao e o HIS, e sim deixar o Keycloak pronto para o HIS autenticar e para a Admin API funcionar.

3. Realm

Para ambiente local simples, pode usar o realm master, que e o padrao atual do HIS.

Se quiser organizar melhor depois, pode criar um realm proprio, por exemplo his.

Para iniciar rapido, use:

Realm: master

4. Criar o client do HIS

  1. Entrar no Keycloak.
  2. Entrar no realm master.
  3. Ir em Clients.
  4. Clicar em Create client.
  5. Preencher:
Client type: OpenID Connect
Client ID: his-web
  1. Salvar.

5. Configurar o client his-web

Depois de criar, configurar o client assim:

Client authentication: ON
Authorization: OFF
Standard flow: ON
Direct access grants: OFF
Implicit flow: OFF
Service accounts roles: ON

Esses pontos sao importantes porque:

  • o HIS faz login web via OIDC
  • o HIS usa client_secret
  • o HIS usa Admin API com client_credentials

6. Configurar roles da service account do client

Depois de salvar o client his-web:

  1. Clicar em Service account roles ou no ponto equivalente da versao atual do Keycloak.
  2. Clicar em Assign role.
  3. Trocar o filtro para Filter by clients.
  4. Adicionar os papeis do client realm-management:
realm-management -> manage-users
realm-management -> view-users

Opcional, mas recomendado:

realm-management -> query-users

Sem isso, a Admin API do HIS pode autenticar, mas falhar ao consultar ou criar usuarios.

7. Configurar URLs do client

No client his-web, configurar:

Root URL: http://localhost:8080
Home URL: http://localhost:8080/ui/home
Valid redirect URIs: http://localhost:8080/login/oauth2/code/keycloak
Opcional para ambiente local mais flexivel: http://localhost:8080/*
Valid post logout redirect URIs: http://localhost:8080/*
Web origins: http://localhost:8080

Para o HIS atual, a URI mais precisa e recomendada e:

http://localhost:8080/login/oauth2/code/keycloak

Se quiser seguir o mesmo padrao mais amplo do seu documento antigo, pode adicionar tambem:

http://localhost:8080/*

8. Copiar o client secret

  1. Abrir o client his-web.
  2. Ir em Credentials.
  3. Copiar o secret gerado.

Esse secret sera usado pelo HIS para:

  • login web OIDC
  • Admin API

9. Criar usuarios no Keycloak

  1. Ir em Users.
  2. Clicar em Add user.
  3. Preencher ao menos:
Username
Email (opcional)
Enabled = ON
  1. Salvar.
  2. Ir em Credentials.
  3. Definir senha.

Observacao: o HIS cria e sincroniza o usuario local pelo keycloakId, mas o login primeiro precisa existir no Keycloak.

10. O que o HIS espera do Keycloak

Ao autenticar, o HIS usa:

  • sub
  • preferred_username
  • email

Depois disso, o usuario ainda precisa de vinculo local no HIS com:

  • unidade
  • perfil

11. Configuracoes do HIS

Essas configuracoes do HIS ja existem no projeto, mas para referencia os valores principais sao:

KEYCLOAK_ISSUER_URI=http://localhost:8081/realms/master
KEYCLOAK_CLIENT_ID=his-web
KEYCLOAK_CLIENT_SECRET=SEU_CLIENT_SECRET
HIS_KEYCLOAK_ADMIN_ENABLED=true
HIS_KEYCLOAK_ADMIN_SERVER_URL=http://localhost:8081
HIS_KEYCLOAK_ADMIN_REALM=master
HIS_KEYCLOAK_ADMIN_CLIENT_ID=his-web
HIS_KEYCLOAK_ADMIN_CLIENT_SECRET=SEU_CLIENT_SECRET

12. Checklist final

  1. Subiu o Keycloak com Docker.
  2. Conseguiu entrar com admin/admin.
  3. Usou o realm master.
  4. Criou o client his-web.
  5. Ligou Client authentication.
  6. Ligou Service accounts roles.
  7. Associou as roles de realm-management para a service account.
  8. Configurou a redirect URI do HIS.
  9. Copiou o secret.
  10. Subiu o HIS.
  11. Testou login em http://localhost:8080.

13. Erros comuns

  • client_credentials recusado: normalmente Service accounts roles esta desligado ou o secret esta errado.
  • 403 ao consultar ou criar usuarios: normalmente faltam roles como manage-users e view-users no client.
  • invalid redirect_uri: a URI http://localhost:8080/login/oauth2/code/keycloak nao foi cadastrada.
  • login funciona, mas o usuario nao entra no HIS: faltam vinculos locais no banco do HIS.
  • realm errado: o HIS esta apontando para um realm diferente.
